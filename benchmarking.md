5. Evaluation and benchmarking
This part describes how we will evaluate the self-supervised model (T-JEPA) trained on the CZI genome-scale CD4+ T-cell Perturb-seq dataset. The model learns, without perturbation labels, a compressed representation ("embedding") of each cell. The evaluation asks one overarching question: does that representation capture real perturbation biology in a way that is useful and generalizable?
The plan has five tasks:
Tasks 1 to 4: decision-driving metrics computed on the training dataset. These tell us whether the representation encodes perturbation effects, and they let us compare model variants (e.g., different regularizers) fairly.
Task 5: an external-dataset transfer test which is the strongest evidence that the model learned generalizable biology rather than dataset-specific structure.
Throughout, every metric is compared against two standard baselines computed on the same data: PCA and scVI. This matters because two recent benchmarks (Ahlmann-Eltze et al., Nature Methods 2025; Bendidi et al. 2024) found that deep-learning foundation models often fail to beat these simple baselines on perturbation tasks. Beating PCA and scVI is therefore the bar that defines success. For scVI, please refer to this github: https://github.com/scverse/scvi-tools
5.0 Background concepts
The embedding: The model maps each cell's expression profile (2000 highly variable genes) to a 256-dimensional vector, the cell embedding. Biologically meaningful structure should be organized in this space.
The perturbation "effect" is a delta: A gene knockdown's effect is not a cell's absolute expression, it is the difference between perturbed cells and matched control cells. So all tasks operate on deltas: (mean of perturbed cells) − (mean of matched control cells), where "matched" means the control reference is built to have the same stimulation-state composition (Rest / Stim8h / Stim48h) as the perturbed group, so the subtraction isolates the perturbation effect rather than a difference in state mix.
Held-out perturbations. To test generalization, certain knockdowns are held out. Importantly, for 5 canonical regulators (CTLA4, FOXP3, CD28, IL2RA, STAT3), the perturbed cells were excluded from model pretraining entirely, so these are strict unseen perturbations (the model never saw them, even unlabeled). These 5 anchor our biological validation. A broader set of additional held-out genes supports the statistical generalization estimate (see "Held-out design" note at the end).
Two decoder strategies: We use a small "decoder" that maps an embedding back to a 2000-gene expression profile. Crucially, this decoder is trained only on control (unperturbed) cells so it never sees a perturbation. So when we feed it a perturbed cell's embedding and it correctly reproduces the perturbation effect, that means the perturbation information was encoded as a geometric shift in a latent space defined entirely by normal-cell biology. That is a strong statement about the representation, and it is the conceptual core of Tasks 1 and 2.
5.1 Task 1 - Decode perturbation effects
Question: Can the embedding be used to predict the transcriptional effect of a gene knockdown?
Procedure:
Train a decoder (a small neural network, "MLP") on 150,000 control cells: input = the cell's 256-d embedding, output = the cell's 2000-gene expression.
For a held-out knockdown (e.g. CD28): encode its perturbed cells into Nx256 matrix with N is the number of perturbed cells of that knockdown. We then average their embeddings Nx256 into one vector 1x256 and decode the vector with the MLP to get the predicted perturbed profile. Do the same for a state-matched control reference → predicted control profile. Subtract predicted perturbed profile to predicted control profile to get predicted effect (the predicted delta).
Compute the true effect directly from the real expression data (mean perturbed − state-matched mean control).
Score the agreement between predicted and true effect: Pearson correlation across the 2000 genes, plus precision@k (of the genes predicted to change most, how many are truly among the most-changed).
Why it matters: This is the most direct test of whether the representation captures perturbation biology. A high score means the model encodes, in its latent geometry, what each knockdown does to the transcriptome, including for knockdowns it never saw. One limitation to keep in mind is that, for the pilot test, we only have 5 held-out genes, so we should analyze the result with caution that this small set might not statistically represent the true effect. 
5.2 Task 2 - Linear vs. non-linear decode (a diagnostic)
Question: Is the perturbation information stored in a simple (linearly readable) form, or a more complex one, or is it absent?
Procedure: Repeat Task 1 with two decoders: a linear one (ridge regression) and the non-linear one (MLP). The deliverable is the gap between them: Pearson score by ridge regression - Pearson score by MLP
Interpretation:
Both high, small gap: information is linearly accessible (ideal; even simple downstream models can use it).
Linear low, MLP high (large gap): information is present but encoded non-linearly (a linear probe can't reach it, the MLP can).
Both low (small gap): information is largely absent or degraded.
Why it matters: A single decode number can mislead. The linear-vs-non-linear gap tells us how the information is stored, which is essential for fairly comparing model variants. For example, one anti-collapse method we are testing (SIGReg) produces a mathematically "spread out" embedding that a linear probe reads poorly even when the information is intact, which means only the gap reveals whether the signal is truly lost or merely non-linearly arranged.
5.3 Perturbation retrieval (is the space organized?)
Question: Independent of decoding, does the embedding space organize perturbations by biology, do knockdowns of functionally related genes land near each other? (This test is essentially close to what we did with BulkFormer’s embedding analysis where we compare the neighbors of certain genes).
Procedure:
For each of the 967 (962 in HVGs, 2 in HVGs AND held-out, 3 not in HVGs BUT held-out) perturbations, compute its effect signature: the mean shift it causes in embedding space (mean perturbed − matched control, kept as a 256-d vector). This gives a 967 × 256 matrix.
Define a relatedness ground truth in advance using curated databases: two genes are "related" if they share, for example, a protein complex (CORUM) or a pathway (Reactome). (We will cap pathway size so that huge generic pathways don't make everything trivially related.)
For each query knockdown, rank all others by similarity of effect signatures and take the top k. Count how many of its pre-defined related partners appear in the top k.
Score with recall@k and mean average precision; compare against PCA/scVI signatures.
A visualization companion (not the metric): We will also project the 967 signatures to 2-D (UMAP) and colour by complex/pathway, to show the organization. But the UMAP is for communication only, UMAP distorts distances and is not comparable across methods, so the quantitative claim should depend on recall@k computed in the full 256-d space.
Why it matters: Decoding tests whether we can read out an effect but retrieval tests whether the space is biologically structured. These reward different geometric properties, so running both gives a balanced assessment. Retrieval is also the natural setting for biologically interpretable queries (see Task 5).
5.4 Effect-size stratification
Question: Is the model genuinely good, or do its averaged scores just reflect that most knockdowns barely perturb the transcriptome?
Background: In genome-scale Perturb-seq, most perturbations have small effects, only ~41% produce a measurable transcriptome-wide change, and the majority of effects are tiny (Replogle et al. 2022). If we average a metric over all 964 knockdowns, the many near-null perturbations dominate and obscure how well the model does on the perturbations that actually do something.
Procedure: Measure each knockdown's true effect magnitude by L2 norm of the ||delta_true|| (size of its real expression delta, and number of differentially expressed genes). Bin the 964 into strong / medium / weak strata. Then re-report every Task 1–3 metric within each stratum.
Interpretation:
The strong stratum is where real biology lives: performance in this bin is the closest thing to an honest "is the model good" number.
The weak stratum is mostly a noise floor (there is little true effect to predict): low scores there are expected and not damning.
A healthy pattern is performance rising with effect size. A flat, uniformly-low pattern would be a warning sign.
The most meaningful baseline comparison is on the strong stratum specifically: can the model predict genuinely-perturbing knockdowns better than PCA?
Why it matters: This is precisely the analysis the recent benchmark papers fault the field for omitting. Reporting stratified results preempts the objection that good averages are inflated by null perturbations, and it tells us where the model is and isn't reliable.
5.5 Cross-dataset transfer
Question: Does the representation transfer to a CD4+ T-cell perturbation dataset the model was never trained on?
External dataset: Arce et al. 2024 (Marson lab, Nature 637:930–939) - Perturb-CITE-seq (CRISPRi knockdown) of primary human CD4⁺ Teff and Treg cells in resting and stimulated states, ~28 knocked-down regulators (MED12, KLF2, MYC, BATF, IRF4, STAT5B, FOXO1, GATA3, etc.). Publicly available (GEO GSE271090; Zenodo 10.5281/zenodo.13924126). It is domain-matched (same cell type, same knockdown modality, rest/stimulated axis) yet genuinely out-of-distribution (different lab, donors, protocol, and a different, much smaller perturbation panel).
Procedure (frozen JEPA encoder):
Align the external dataset's genes to our 2000-HVG vocabulary; report the overlap (a low overlap would confound the test).
Encode the external cells through the frozen CZI-trained JEPA encoder.
Fit a fresh decoder on the external dataset's control cells (so the test isolates the encoder's transferability, not the decoder's).
Run the Task 1 (decode) and Task 3 (retrieval) evaluations on the external data.
The key test - cross-dataset agreement: for knockdowns present in both datasets (the overlap of our 964 with their ~28 - likely MED12, MYC, BATF, IRF4, STAT5B, FOXO1, GATA3…), compute each gene's effect signature independently in each dataset (both in our latent space) and measure whether they point the same direction. If knocking down MED12 produces the same embedding-space shift in both datasets, the model captured something real and reproducible about MED12's effect that survived a complete change of lab and protocol.
The unavoidable confounding: Two datasets differ technically (protocol, sequencing depth, batch), so a drop in transfer performance could reflect those differences rather than a failure of biology. We manage this by (a) pairing transfer scores with a batch-integration diagnostic, (b) framing results as "transfer under domain shift" (a positive result is strong precisely because it survived the shift), and (c) leaning on the cross-dataset agreement test, which is a relative measure where dataset-level offsets largely cancel.
Interpretation: Transfer scores will be lower than in-distribution scores. The meaningful result is whether the model degrades less than PCA/scVI, and whether the same knockdown's effect is reproducible across datasets (matched-gene agreement much higher than random-gene-pair agreement, and higher for our model than for the baselines).
Why it matters: In-distribution performance can reflect dataset-specific memorization. Surviving a transfer to an independent lab's data is the strongest evidence that the model learned generalizable T-cell perturbation biology.
All five are run identically on PCA and scVI baselines. The model earns its keep only by beating those, and the most informative comparisons are on strong-effect perturbations (Task 4) and on cross-dataset agreement (Task 5).
